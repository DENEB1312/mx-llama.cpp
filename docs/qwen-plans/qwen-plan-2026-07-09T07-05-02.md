## 1. Mathematical Formulation

The per-token recurrence (matches `gated_delta_net.cu:92-103` non-KDA and `:113-122` KDA):

```
S_t[i,c] = g_t[i] * S_{t-1}[i,c] + k_t[i] * β_t * (v_t[c] - Σ_j g_t[j] * S_{t-1}[j,c] * k_t[j])
o_t[c]   = Σ_i S_t[i,c] * q_t[i]
```

For a chunk of CS tokens indexed `i, j ∈ [0, CS)`, define cumulative gate `G[i] = Σ_{t≤i} g_t` and decay matrix `D[i,j] = exp(G[j] - G[i])` for `j ≥ i` (lower-triangular, `delta-net-base.cpp:120-124`).

**Intra-chunk system.** Let `L[i,j] = D[i,j] · k[i]·k_b[j]` for `j < i` (strictly lower), where `k_b = k · β`. The chunk-internal "attention" matrix `A` solving the implicit recurrence satisfies:

```
(I + L) · A = -L        ⟹   A = -(I + L)⁻¹ · L = (I + L)⁻¹ - I
```

This is `delta-net-base.cpp:162-172`: `lin_solve = solve_tri(I+L, -L)` then `A = lin_solve + I`. The intra-chunk output contribution is `v_intra = Aᵀ · v_b` (`:175`).

**Inter-chunk state.** After solving the chunk, the state update at chunk end is:

```
S_new = S_old · exp(G[CS-1])  +  K_gᵀ · v_new
```

where `K_g[i, j] = k[i] · exp(G[CS-1] - G[j])` (`delta-net-base.cpp:201-209`) and `v_new = v - K_cdᵀ · S_old` with `K_cd = (K_b · exp(G))ᵀ · A` (`:183`). The per-chunk output combines `o = S_old · q_g_exp + v_new · K_q` (`:235-241`).

Crucially, the state update is **sequential across chunks** (`delta-net-base.cpp:216-244`), so chunks within one (head, seq) must be processed in order. Chunks across different (head, seq) are fully independent.

## 2. Kernel Architecture

**Recommended: single fused kernel per (head, seq), looping over chunks internally.** Rationale: avoids writing per-chunk intermediates (each `CS × S_v × H × n_seqs`) back to global memory between an "intra" and "inter" kernel. The state `S_v × S_v` stays in registers/shared across chunks.

### Grid and block

```
gridDim  = (H, n_seqs, 1)
blockDim = (warp_size, num_warps_y, 1)
```

Where `warp_size = 64` on gfx906, and `num_warps_y` is chosen so that `warp_size · num_warps_y ≥ S_v` (one lane-y per state row for column-parallel work). For `S_v=128`: `num_warps_y = 2` gives 128 lane-y positions. For `S_v=16`: `num_warps_y = 1` and lanes 16..63 are masked during state ops.

### Thread roles

| Phase | threadIdx.x role | threadIdx.y role |
|---|---|---|
| Load chunk (k, q, v, g, β) | position within CS (mod CS) | feature row of S_v |
| g_cs cumsum | parallel-scan within wavefront | — |
| Build L (CS×CS) | row i of L | column j of L (or iterated) |
| Forward solve (I+L)x=b | row i of solution | — |
| State update S_v×S_v | column of state | row of state |
| Output write | position within CS | feature row |

For CS=64 (non-KDA) the `threadIdx.x` lane maps 1:1 to a chunk position. For CS=16 (KDA) each chunk position is owned by lane `pos` with lanes `pos+16, pos+32, pos+48` idle or reassigned to different sub-tasks — handled via `lane < CS` guards.

### Shared memory budget (gfx906 has 64 KB LDS/CU)

| Buffer | Size (worst case) |
|---|---|
| `k_chunk[CS][S_v]` | 64 × 128 × 4 = 32 KB |
| `v_chunk[CS][S_v]` | 32 KB |
| `L[CS][CS]` + `A[CS][CS]` | 2 × 64² × 4 = 32 KB |
| `g_cs[CS]`, `beta[CS]`, scalars | <1 KB |

Total ≈ 97 KB — **oversubscribes LDS**. Mitigation: never all resident simultaneously. `k_chunk` and `v_chunk` can be streamed through registers (each lane owns `rows_per_lane` features, same pattern as `gated_delta_net.cu:53`), and only the CS×CS matrices `L`/`A` live in LDS. Realistic LDS: ~32 KB for `L`+`A`+misc, well within budget.

## 3. Pseudocode

```
kernel gated_delta_net_chunked<S_v, KDA, CS>(q, k, v, g, beta, state_in, dst, ...):
    (h, seq) = (blockIdx.x, blockIdx.y)
    lane = threadIdx.x;  row_y = threadIdx.y
    load state shard into registers (same layout as per-token kernel, lines 53-58)

    for each chunk c in [0, n_chunks):
        // === LOAD ===
        for each local position p in chunk owned by this lane:
            load k[p][*], q[p][*], v[p][*], g[p], beta[p] into registers / smem
        __syncthreads()

        // === CUMSUM g → g_cs ===
        parallel-scan across lanes (CS elements, warp shuffle)
        g_last = g_cs[CS-1]   (broadcast)

        // === BUILD L (strictly lower, CS×CS, in smem) ===
        for (i, j) assigned to (lane, row_y) with j < i:
            if KDA:
                L[i,j] = exp(g_cs[j] - g_cs[i]) * dot(k[i][:], k_b[j][:])   // dot over S_v
            else:
                L[i,j] = exp(g_cs[j] - g_cs[i]) * dot(k[i][:], k_b[j][:])
        __syncthreads()

        // === FORWARD SOLVE (I + L) x = -L, column by column ===
        for col = 0 .. CS-1:
            if lane == col:
                // x[0..col-1] already solved; compute x[col]
                acc = -L[col, col]  (which is 0 since strictly lower, so rhs = 0 on diag)
                for r < col: acc -= L[col, r] * A[r, col]
                A[col, col] = acc        // (I+L) has 1 on diagonal → no division
                A[col, col] += 1          // final A = solve + I
            else if lane > col:
                // compute A[lane, col] similarly
                acc = -L[lane, col]
                for r < col: acc -= L[lane, r] * A[r, col]
                A[lane, col] = acc
                if lane == col: A[lane, col] += 1
            __syncwarp()   // each column depends on previous columns within warp
        __syncthreads()

        // === INTRA-CHUNK v_new = v - Aᵀ · v_b; o_intra = Aᵀ · v_b contribution ===
        for each position p owned by lane:
            v_new[p][:] = v[p][:] - Σ_j A[j, p] * v_b[j][:]    // sum over CS
            o_intra[p][:] = Σ_j A[j, p] * v_b[j][:]

        // === OUTPUT o[p] = s_oldᵀ · (q[p] · exp(g_cs[p])) + o_intra[p] ===
        for each (p, feature) owned by (lane, row_y):
            inter = dot_s_state_col(s_shard, q_g_exp[p])   // warp-reduce
            o[p][feature] = inter + o_intra[p][feature]
        write o chunk to dst

        // === STATE UPDATE: s = s · exp(g_last) + k_gᵀ · v_new ===
        for each state element s[i, col] owned by (lane=row_y, col=lane or via y):
            s[i, col] *= exp(g_last)
            s[i, col] += Σ_p k_g[p][i] * v_new[p][col]     // sum over CS
        __syncthreads()

    write final state to dst state region (matches per-token lines 150-155)
```

Key subtleties:
- The forward solve is **sequential across columns** (`col` loop), but embarrassingly parallel across rows `> col` within a column. With CS=64 and warp_size=64, one wavefront covers all rows and `__syncwarp()` between columns is sufficient (no LDS round-trip).
- For CS=16 (KDA) the same logic works; lanes 16..63 are idle in the solve — cheap because CS² is tiny (256 ops).

If the chunked kernel produces NaN/Inf (detectable via a debug flag or `GGML_CUDA_CHECK`), the dispatch should silently fall back to the per-token kernel. Add a `cparams` flag `use_chunked_gdn` defaulting to `true` so users can disable it.

## 6. Testing Strategy

| Level | What | How |
|---|---|---|
| **Unit: solve_tri** | `A = (I+L)⁻¹·(-L+I)` is correct | Generate random strictly-lower `L` on host, compute reference with a CPU forward-substitution, compare kernel output. Tolerance: `1e-4` relative for FP32. |
| **Unit: single chunk** | Chunked kernel with `n_tokens = CS` matches per-token kernel run on same input | Set `n_tokens = 64` (or 16 for KDA), identical initial state, compare `o` and final `S`. |
| **Regression: multi-chunk** | `n_tokens ∈ {CS, 2·CS, 5·CS, 137}` (non-multiple of CS exercises padding path) | Run both kernels, assert `max_abs_diff < 1e-3` and `mean_abs_diff < 1e-5`. |
| **Edge cases** | `n_tokens < CS` (should route to per-token), `n_tokens == 1` (AR path), `keep_rs_t == true` (must not enter chunked) | Explicit asserts in dispatch + integration tests. |
| **End-to-end** | DeltaNet-base model perplexity on a held-out chunk with `cparams.fused_gdn_ch = true` | Must match CPU chunked path within 0.1% PPL. |
| **Performance** | pre-fill throughput (tokens/sec) for `n_tokens ∈ {128, 512, 2048}` | Expect ≥2× speedup over per-token kernel at 2048 tokens on MI50. |

Concrete test harness: add a case to `tests/test-backend-ops.cpp` mirroring the existing `gated_delta_net` test but with `n_tokens > 1` and the fused op enabled. The CPU `build_delta_net_chunking` path (`delta-net-base.cpp:23`) is the ground truth.

## 7. Ordered Phases

1. **Phase 1 — Solve-tri micro-kernel** (S): implement and unit-test the `(I+L)·A = -L+I` forward solve on device. Shippable: a standalone test binary.
2. **Phase 2 — Single-chunk kernel** (M): implement kernel for `n_tokens ≤ CS` only. Validate against per-token kernel. Rollback: dispatch unchanged.
3. **Phase 3 — Multi-chunk loop** (M): add the chunk loop with state propagation. Validate regression.
4. **Phase 4 — Dispatch integration + K>1 snapshot support** (M): wire into `launch_gated_delta_net`, add `keep_rs_t=true` variant or document as unsupported.
5. **Phase 5 — Performance tuning** (S): tune block dim, LDS usage, `__launch_bounds__` occupancy. Profile with rocprof.

Each phase is independently testable and revertable.
