## 1. ngrok

**How it works:** ngrok runs an agent on your machine that establishes an outbound TLS connection to ngrok's cloud infrastructure. The cloud edge then routes public URLs (HTTP, TCP, UDP, or SSH) back through that tunnel to your localhost. You point your iPhone at the ngrok-provided URL.

**Free tier details:**
- 1 HTTP endpoint (dynamic subdomain, changes on restart)
- 100 MB of traffic per month (as of mid-2025; previously unlimited)
- No custom domains on free tier
- Auto-reconnect and session history limited
- Bandwidth capped at ~1 Mbps effective throughput

**Setup:**
```bash
# Install
brew install ngrok        # macOS
# or download from https://ngrok.com

# Authenticate (create free account, get authtoken)
ngrok config add-authtoken YOUR_TOKEN

# Expose a local web server on port 3000
ngrok http 3000

# Or expose SSH directly
ngrok tcp 22
```

**iOS compatibility:** Excellent. The iPhone simply opens the ngrok HTTPS URL in Safari or uses it as a base URL in any app. For TCP/SSH tunnels, you can use third-party SSH clients (Termius, Blink Shell) to connect through the ngrok TCP endpoint. No native iOS app needed for basic HTTP tunneling.

**Pros:**
- Easiest setup of any option; single binary
- Well-documented with web dashboard and CLI
- Supports HTTP, TCP, TLS, WebSocket
- Web inspection UI (localhost:4040) for debugging requests
- Works behind NAT and most firewalls

**Cons:**
- Free tier traffic cap is now restrictive (100 MB/mo)
- Dynamic subdomain; no persistent URL without paid plan
- Connection drops if agent restarts (though auto-reconnect exists)
- Visible ngrok branding in some contexts

## 2. Cloudflare Tunnel (cloudflared)

**How it works:** `cloudflared` creates an outbound connection to Cloudflare's edge network. Traffic from the public internet enters at Cloudflare's nearest PoP and is routed through the tunnel to your localhost. You can expose HTTP services, TCP services, or full domains.

**Free tier details:**
- **Unlimited bandwidth** — no traffic cap
- Free DNS zone (yourname.trycloudflare.com) for quick testing
- Requires a Cloudflare account (free)
- For persistent custom domains, you need to verify domain ownership via DNS (also free)
- No hard rate limits on the free tier

**Setup:**
```bash
# Install
brew install cloudflared        # macOS
sudo apt install cloudflared    # Ubuntu/Debian

# Quick HTTP tunnel (no domain needed — generates a random .trycloudflare.com URL)
cloudflared tunnel --url http://localhost:3000

# Persistent tunnel (requires Cloudflare account + domain or quick trial domain)
cloudflared tunnel create my-tunnel
cloudflared tunnel route dns my-tunnel myapp.trycloudflare.com
cloudflared tunnel run my-tunnel
```

**iOS compatibility:** Excellent. The iPhone accesses the public HTTPS URL via Safari or any app. Since Cloudflare terminates TLS at the edge, your iPhone sees a valid certificate (even on the free trycloudflare.com domain).

**Pros:**
- Truly free with no bandwidth cap — best free tier available
- Uses Cloudflare's global CDN (good performance worldwide)
- Free DDoS protection built in
- Supports custom domains for free (you just need to own a domain, ~$10/yr)
- Enterprise-grade security and reliability

**Cons:**
- Slightly more setup than ngrok (especially for persistent tunnels)
- Requires a Cloudflare account
- The `--url` one-liner generates random subdomains that change on restart unless you set up a proper tunnel + DNS record
- Less granular traffic inspection UI compared to ngrok's dashboard

## 3. FRP (Fast Reverse Proxy / frp)

**How it works:** frp is an open-source, self-hosted reverse proxy. You run a `frps` server (the frp server component) on a cloud VPS with a public IP, and a `frpc` client (the frp client) on your localhost. The client connects to the server over TCP or WebSocket, and the server forwards public traffic back through that connection.

**Free tier details:**
- **Completely free** — no tiers, no caps, no accounts
- You must provide and maintain your own VPS (the "free" part is the software; infrastructure costs money unless you use a free-tier cloud instance)
- Some providers offer free VPS tiers (Oracle Cloud Always Free, Google Cloud free tier e2-micro in select regions), but these have limitations

**Setup:**
```bash
# On your VPS (frps server)
wget https://github.com/fatedier/frp/releases/download/v0.61.1/frp_0.61.1_linux_amd64.tar.gz
tar -xzf frp*.tar.gz
cat > frps.toml << EOF
bindPort = 7000
authenticationMethod = token
token = your-secret-token
EOF
./frps -c frps.toml

# On your localhost (frpc client)
cat > frpc.toml << EOF
serverAddr = "your-vps-ip"
serverPort = 7000
authenticationMethod = token
token = your-secret-token

[[proxies]]
name = "web"
type = "tcp"
localIP = "127.0.0.1"
localPort = 3000
remotePort = 9000
EOF
./frpc -c frpc.toml

# Access from iPhone: http://your-vps-ip:9000
```

**iOS compatibility:** Good, but depends on what you expose. For HTTP services, the iPhone just visits the VPS IP + port. For SSH access, you can set up a separate TCP proxy for port 22 and use an SSH client on iOS. Note that exposing raw IPs to the internet without TLS means traffic is unencrypted unless you add TLS termination (e.g., with Caddy or Nginx in front).

**Pros:**
- Full control — you own everything
- No vendor lock-in, no usage limits
- Supports many protocols: TCP, UDP, HTTP, HTTPS, STCP, SUDP, XTCP
- P2P mode (XTCP) can bypass the server for direct connections
- Highly configurable with TOML/YAML config
- Active community and frequent updates

**Cons:**
- Requires a VPS — not truly "free" unless you already have one
- You handle security, TLS, certificates, and maintenance
- More complex setup than ngrok or cloudflared
- No built-in web dashboard (though frps has a simple stats page)
- Self-hosting responsibility (updates, monitoring, backups)

## 4. Serveo

**How it works:** Serveo runs on top of OpenSSH. You connect to a shared Serveo server via SSH with port forwarding flags, and the server exposes your local ports as public URLs. It's essentially SSH tunneling as a service.

**Free tier details:**
- Completely free, no accounts needed
- **Shared subdomains** — you get a random hostname like `abc123.serveo.net`
- Connection timeout: idle connections are dropped after ~15 minutes; sessions max out around 4 hours
- Bandwidth is reasonable but not guaranteed
- No custom domains on free tier

**Setup:**
```bash
# Expose HTTP on port 3000
ssh -R 80:localhost:3000 serveo.net

# With a custom subdomain (if available)
ssh -R 80:localhost:3000 serveo.net -N -R yourname.serveo.net:80:localhost:3000

# Expose SSH on port 22
ssh -R 2222:localhost:22 serveo.net

# Keep alive (required — Serveo kills idle connections)
ssh -o ServerAliveInterval=60 -o ServerAliveCountMax=3 -R 80:localhost:3000 serveo.net
```

**iOS compatibility:** Moderate. Serveo itself provides HTTP URLs accessible from any browser on iPhone. However, the underlying mechanism is SSH-based, which means you can also use it to forward an SSH session and then connect from iOS via Termius or Blink Shell. The main limitation is that connections are ephemeral — they time out.

**Pros:**
- Zero setup for the server side — just SSH (which macOS and many Linux distros have built-in)
- No installation required on your machine beyond SSH client
- Simple one-liner commands
- Works with any port forwarding SSH supports

**Cons:**
- Connections are unreliable — time out after ~15 min idle, max ~4 hours per session
- Shared subdomains (no persistence)
- Server is community-maintained and can be slow or unreliable
- No dashboard or traffic inspection
- Not suitable for anything requiring sustained uptime
- May be blocked by some networks/firewalls

## 6. Tailscale Funnel (SSH-based alternative)

**How it works:** Tailscale runs a WireGuard-based mesh network between devices. "Funnel" is a feature that exposes a Tailscale service to the public internet via Cloudflare's edge. It's not exactly ngrok-style, but it achieves similar goals with different tradeoffs.

**Free tier details:**
- Free for personal use (up to 3 users, 100 devices)
- Unlimited bandwidth within the Tailscale network
- Funnel feature is free
- Requires Tailscale account

**Setup:**
```bash
# Install on Mac and iPhone
brew install tailscale
sudo tailscale up

# On your Mac, expose a local service via Funnel
tailscale funnel http :3000

# Access from iPhone browser at the generated URL
```

**iOS compatibility:** Best-in-class. Since Tailscale has an official iOS app, you get seamless integration — no public URLs needed if both devices are on the same Tailscale network. With Funnel, the public URL works in any browser.

**Pros:**
- Official, well-maintained iOS app
- End-to-end encrypted (WireGuard)
- No bandwidth limits
- Works great for personal/team use without exposing to the entire internet
- Funnel provides public access when needed

**Cons:**
- Funnel routes through Cloudflare, adding a hop
- Requires Tailscale account and auth key setup on each device
- Slightly more complex than ngrok for one-off sharing
- Not designed as a general-purpose reverse proxy (it's primarily a mesh network)

## 7. SSH Reverse Tunneling (DIY with your own VPS)

**How it works:** If you have any VPS (even a cheap $5/mo one), you can use raw SSH to create reverse tunnels. This is the most "pure" SSH-based approach and gives you full control.

**Setup:**
```bash
# On your Mac, create a reverse tunnel through SSH
ssh -R 0.0.0.0:8080:localhost:3000 user@your-vps.com

# This forwards VPS:8080 → localhost:3000

# For persistent tunnels, use autossh:
autossh -M 0 -o "ServerAliveInterval=30" -o "ServerAliveCountMax=3" \
  -R 0.0.0.0:8080:localhost:3000 user@your-vps.com

# Add TLS with a reverse proxy on the VPS (Caddy is easiest):
# Caddyfile:
# your-domain.com {
#     reverse_proxy localhost:8080
# }
```

**iOS compatibility:** Excellent. The iPhone accesses `https://your-domain.com` (with proper TLS) or `http://your-vps-ip:8080`. Works in any browser.

**Pros:**
- Full control and customization
- Can add TLS, authentication, rate limiting
- No vendor lock-in
- One VPS can serve many tunnels

**Cons:**
- Requires a VPS (costs money)
- You manage security patches, certificates, monitoring
- More complex than managed solutions

## Recommendation

**For most iPhone-access use cases, Cloudflare Tunnel (`cloudflared`) is the best free option.** It offers unlimited bandwidth, built-in TLS, persistent URLs (with a domain or even the free `trycloudflare.com` subdomain), and runs on any platform. The setup is straightforward:

```bash
# Quick one-liner for testing:
cloudflared tunnel --url http://localhost:3000

# For something more permanent (free, no monthly cost):
cloudflared tunnel create my-tunnel
cloudflared tunnel route dns my-tunnel myapp.trycloudflare.com
cloudflared tunnel run my-tunnel
```

**If you already have a VPS**, the DIY SSH + Caddy approach gives you maximum flexibility at minimal ongoing cost (~$5/mo for a basic VPS, domain ~$10/yr).

**If you want the absolute simplest experience** and don't mind the 100 MB/month cap (or are willing to pay $5/mo for ngrok's paid plan), ngrok remains the most polished tool with the best developer experience.

**If both your Mac and iPhone are on Tailscale**, use Tailscale Funnel — it provides encrypted, zero-config access with an excellent iOS app integration, and is genuinely free for personal use.

## Recommendation

**For most iPhone-access use cases, Cloudflare Tunnel (`cloudflared`) is the best free option.** It offers unlimited bandwidth, built-in TLS, persistent URLs (with a domain or even the free `trycloudflare.com` subdomain), and runs on any platform. The setup is straightforward:

```bash
# Quick one-liner for testing:
cloudflared tunnel --url http://localhost:3000

# For something more permanent (free, no monthly cost):
cloudflared tunnel create my-tunnel
cloudflared tunnel route dns my-tunnel myapp.trycloudflare.com
cloudflared tunnel run my-tunnel
```

**If you already have a VPS**, the DIY SSH + Caddy approach gives you maximum flexibility at minimal ongoing cost (~$5/mo for a basic VPS, domain ~$10/yr).

**If you want the absolute simplest experience** and don't mind the 100 MB/month cap (or are willing to pay $5/mo for ngrok's paid plan), ngrok remains the most polished tool with the best developer experience.

**If both your Mac and iPhone are on Tailscale**, use Tailscale Funnel — it provides encrypted, zero-config access with an excellent iOS app integration, and is genuinely free for personal use.
